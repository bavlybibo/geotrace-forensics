# Release Checklist

Use this checklist before publishing a public release.

## Automated gates

- [ ] Run `python -m compileall -q app tests main.py`
- [ ] Run `python -m pytest -q` on Windows
- [ ] Run `build_windows_exe.bat`
- [ ] Confirm production build uses `geotrace_forensics_x.spec` and excludes `demo_evidence`
- [ ] Confirm demo build, if needed, uses `geotrace_forensics_x_demo.spec`
- [ ] Confirm `release\SHA256SUMS.txt` is generated by `make_release.bat`

## Manual EXE smoke test

- [ ] Launch `dist\GeoTraceForensicsX\GeoTraceForensicsX.exe`
- [ ] Create/open a case
- [ ] Import at least one image
- [ ] Run quick analysis
- [ ] Run OCR/map analysis with and without Tesseract installed
- [ ] Open Dashboard, Evidence Queue, Geo page, AI Guardian, and Reports
- [ ] Export Internal Full package
- [ ] Export Shareable Redacted package
- [ ] Export Courtroom Redacted package
- [ ] Verify redacted/courtroom packages do not include `chart_map.png` or `geolocation_map.html`
- [ ] Run Courtroom Package Verifier and confirm expected status
- [ ] Test backup creation and restore
- [ ] Confirm no real evidence is present in `case_data`, `exports`, `logs`, `.temp`, `build`, or `dist` before source publishing

## Documentation gates

- [ ] README version and `APP_VERSION` match
- [ ] README channel and `APP_BUILD_CHANNEL` match
- [ ] LICENSE exists
- [ ] PRIVACY.md exists
- [ ] SECURITY.md exists
- [ ] DISCLAIMER.md exists
- [ ] THIRD_PARTY_NOTICES.md exists
- [ ] Real screenshots are added under `screenshots/`
- [ ] README screenshot links render correctly on GitHub
